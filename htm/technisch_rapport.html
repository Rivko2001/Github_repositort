Technisch rapport website

Eindversie van het technische rapport over het ontwerp, de opbouw en het werkproces van mijn website.

Bestandsstructuur

Voor dit project heb ik mijn bestanden opgedeeld in voor mij logische mappen. Zo heb ik aparte mappen gemaakt voor CSS, data, HTML, images en JavaScript. Daarnaast staan het logo en de index in de hoofdmap. Deze structuur heb ik bewust gekozen om overzicht te houden en sneller bestanden terug te kunnen vinden.

Binnen de image-map heb ik eerst alle afbeeldingen bij elkaar gezet, maar later heb ik dit verder gestructureerd. Zo heb ik per stad aparte mappen gemaakt. Dit maakt het overzichtelijker en zorgt ervoor dat ik makkelijker nieuwe afbeeldingen kan toevoegen zonder dat alles door elkaar komt te staan.

In eerste instantie was mijn plan om ongeveer twintig steden toe te voegen. Uiteindelijk heb ik ervoor gekozen om me te focussen op kwaliteit in plaats van kwantiteit. Daarom heb ik drie steden uitgewerkt, zodat het concept van mijn website duidelijk en goed uitgewerkt is.

HTML

Voor de HTML heb ik gewerkt met meerdere pagina’s. Ik heb een indexpagina gemaakt die dient als startpunt van de website. Op deze pagina staat een binnenkomend filmpje en daaronder een kaart met de steden die ik heb bezocht. Deze kaart heb ik zo opgezet dat ik later eenvoudig nieuwe steden kan toevoegen.

Daarnaast heb ik voor elke stad een aparte HTML-pagina gemaakt. Dit zorgt voor overzicht en maakt het makkelijker om per stad content aan te passen of uit te breiden.

Binnen de HTML heb ik gewerkt met logische blokken. Elk blok bestaat uit een combinatie van tekst en een afbeelding. Dit heb ik gedaan zodat ik makkelijk blokken kan toevoegen, verwijderen of aanpassen zonder dat de hele pagina opnieuw opgebouwd moet worden. Op de indexpagina wisselen de afbeeldingen en tekst elkaar af, terwijl op de andere pagina’s de structuur consistenter is gehouden.

CSS

Voor de CSS had ik in het begin het idee om voor elk onderdeel een apart CSS-bestand te maken. Dit leek mij logisch, omdat ik in eerdere projecten had gemerkt dat één groot CSS-bestand snel onoverzichtelijk wordt en dat er veel dubbele code in komt te staan.

Tijdens het werken aan dit project liep ik echter tegen een probleem aan. De verschillende CSS-bestanden gingen elkaar soms overschrijven. Dit kwam doordat de volgorde waarin CSS wordt ingeladen bepaalt welke stijl uiteindelijk wordt toegepast. De onderste regel wint, waardoor eerdere stijlen soms niet meer werkten zoals bedoeld.

Daarom heb ik uiteindelijk besloten om alles samen te voegen in één CSS-bestand. Dit zorgde voor meer overzicht en minder fouten. Vervolgens heb ik AI gebruikt om mijn CSS te laten controleren en dubbele of overbodige regels eruit te halen. Dit hielp mij om de code schoner en efficiënter te maken.

JavaScript en data

Voor JavaScript heb ik meerdere onderdelen gebruikt. Zo heb ik JavaScript ingezet voor de interactieve kaart, maar ook voor de pizza-knop op mijn website. In het begin had ik alleen een pizza, maar later heb ik dit uitgebreid met pasta, waardoor het een wisselende knop is geworden. Dit zorgt voor meer interactie op de pagina.

Daarnaast heb ik JavaScript gebruikt voor mijn datavisualisatie. Bij het opslaan van mijn data heb ik ervoor gekozen om de gegevens per stad te structureren in plaats van per onderwerp. Dit betekent dat alle informatie van één stad bij elkaar staat, zoals inwoners, toeristen en oppervlakte. Ik heb hier bewust voor gekozen omdat dit beter aansluit bij de opbouw van mijn website. Elke stad heeft namelijk een eigen pagina, waardoor het logischer en overzichtelijker is om alle gegevens per stad te groeperen. Een andere mogelijkheid was geweest om de data per onderwerp te structureren, bijvoorbeeld alle inwoners van alle steden bij elkaar. Dit kan handig zijn voor datavisualisaties, maar maakt het lastiger om nieuwe steden toe te voegen. Door mijn keuze kan ik eenvoudig een nieuwe stad toevoegen door één blok data toe te voegen

De pizza-knop heb ik zelf geschreven. Voor de rating-knop met effecten zoals confetti en bewegende smileys heb ik AI gebruikt om een basis te genereren. Daarna heb ik steeds gevraagd wat de code precies doet, zodat ik het ook begrijp en eventueel zelf kan aanpassen.

Rating (JavaScript)

Voor het onderdeel rating heb ik gebruik gemaakt van JavaScript om interactie toe te voegen aan mijn website. Gebruikers kunnen per stad een beoordeling geven op verschillende onderdelen, zoals sfeer en eten.

Het is belangrijk om te vermelden dat deze ratings niet worden opgeslagen. Als ik gebruik had gemaakt van een server, had ik de beoordelingen kunnen opslaan in een database. Op die manier had ik bijvoorbeeld per stad een gemiddelde score kunnen tonen, gebaseerd op alle gebruikers.

Omdat ik geen server heb gebruikt in dit project, blijven de ratings alleen lokaal op de pagina zichtbaar en worden ze niet bewaard. De functionaliteit is dus vooral bedoeld om te laten zien hoe interactieve elementen werken binnen een website.

De andere JavaScript-bestanden op mijn website werken op een vergelijkbare manier. Ze zorgen voor interactie binnen de pagina zelf, maar slaan geen gegevens op en communiceren niet met een server.

Leaflet (webmaps)

Voor de kaarten op mijn website heb ik gebruik gemaakt van Leaflet. Op elke stedenpagina staat een kaart met de tien leukste plekken om te bezoeken. Hierbij heb ik ook een legenda toegevoegd, zodat duidelijk is wat de verschillende punten betekenen.

Deze legenda staat bewust boven de kaart, zodat deze goed zichtbaar blijft tijdens het gebruik. Ik heb ervoor gekozen om de kaart rustig en overzichtelijk te houden door een lichte kaartstijl te gebruiken. Hierdoor vallen de punten beter op en blijft de kaart duidelijk.

Testen en optimalisatie

Voor de prestaties van mijn website heb ik mijn afbeeldingen verkleind tot onder de 1 MB. Dit heb ik gedaan om te zorgen dat de website sneller laadt.

Daarnaast heb ik mijn website getest op verschillende apparaten. Met behulp van de F12-tool in Google Chrome heb ik de website bekeken op mobiel, tablet en laptop. Hierbij heb ik gecontroleerd of de layout goed werkt en waar nodig aangepast.

GitHub

Voor versiebeheer heb ik gebruik gemaakt van GitHub. Hiermee kon ik mijn werk opslaan en delen met de docent. Daarnaast kon ik mijn voortgang bijhouden en voorkwam ik dat alles alleen lokaal opgeslagen werd.

Gebruik van AI

Tijdens dit project heb ik gebruik gemaakt van AI, zoals ChatGPT. AI heeft mij geholpen bij het oplossen van problemen en het verbeteren van mijn code, bijvoorbeeld bij interactieve onderdelen en het opschonen van CSS.

Ik heb AI niet blind gebruikt, maar altijd geprobeerd te begrijpen wat de code doet door om uitleg te vragen. Dit vond ik belangrijk, zodat ik zelf controle hield over mijn website.

Een nadeel van AI was dat het niet altijd in één keer goed werkte. Soms moest ik meerdere keren mijn vraag aanpassen voordat ik een goed resultaat kreeg. Hierdoor bleef het belangrijk om kritisch te blijven en niet alles direct over te nemen.

Zonder AI had mijn website er waarschijnlijk minder goed uitgezien en had ik meer tijd nodig gehad om bepaalde onderdelen werkend te krijgen.

Afsluiting

Dit project heeft mij geholpen om beter te begrijpen hoe een website wordt opgebouwd en hoe verschillende technieken samenwerken. In de toekomst zou ik dit willen uitbreiden met een backend, zodat functionaliteiten zoals de rating ook daadwerkelijk opgeslagen kunnen worden.
